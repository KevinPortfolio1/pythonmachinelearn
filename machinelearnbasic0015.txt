機器學習 小到大

| 資料量大小 | 特徵           | 對機器學習的影響            |
| ----- | ------------ | ------------------- |
| 少量資料  | 資料有限、容易過擬合   | 需使用簡單模型、正則化、交叉驗證等技巧 |
| 中等資料  | 資料夠用，但可能不夠全面 | 表現較穩定，可嘗試更複雜模型      |
| 大量資料  | 涵蓋更多變異、降低偏差  | 可使用深度學習、訓練效果通常更好    |

2. 資料如何從小變大？

如果你是問如何讓資料「從小變大」——資料擴增的技巧：

✅ 數據擴增（Data Augmentation）：特別常用於圖像、語音、文字等非結構化資料。

✅ 合成資料生成（Synthetic Data）：使用 GAN、模擬器或 LLM 生成人造資料。

✅ 網路爬蟲、API 收集：擴充真實世界資料。

✅ 半監督學習 / 自監督學習：使用大量未標註資料提升模型表現。

✅ 轉移學習 + 微調：當資料少時，用大模型作為起點。

3. 是否在問「資料前處理：小資料轉成大維度特徵」？

這可能是指從簡單的原始資料萃取出更多的特徵，例如：

NLP：從單詞擴展成向量（如 word2vec, BERT embedding）

Tabular Data：特徵工程（衍生變數、分箱、轉換）

Image：圖像經過卷積神經網路後轉成高維度特徵圖

機器學習與資料規模的實務建議：

| 資料量 | 建議模型與技術                   |
| --- | ------------------------- |
| 很少  | 線性模型、SVM、小樹、KNN           |
| 中等  | 隨機森林、XGBoost、簡單神經網路       |
| 很大  | CNN、RNN、Transformer、預訓練模型 |


1. 線性模型（Linear Models）

常見：Logistic Regression、Linear Regression、Ridge、Lasso

🔹 2. 支援向量機（SVM）

包含線性SVM與非線性（核方法SVM）


🔹 3. 小決策樹（Decision Tree）

簡單可解釋的樹狀模型，適合少量特徵或需要解釋的場景

🔹 4. K 最近鄰（K-Nearest Neighbors, KNN）

基於距離的分類（或回歸）演算法
